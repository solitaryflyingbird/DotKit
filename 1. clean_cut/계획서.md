# 2D 게임 그래픽 리소스 선따기 자동화 도구 기획

## 배경 및 문제점

2D 게임 그래픽 제작에서 리소스 선따기(스프라이트 추출, 배경 제거)는 단순 반복적이면서도 높은 정밀도가 요구되는 작업이다. 특히 다음과 같은 문제가 발생한다:

- 안티앨리어싱으로 인한 경계 픽셀의 반투명/회색 잔여물
- JPEG 등 손실 압축 포맷의 노이즈로 인해 순수 흰색이 아닌 픽셀 발생
- 수작업 보정에 과도한 시간 소요

## 핵심 아이디어

### 1단계: 배경 영역 탐색 및 삭제

- 흰색 배경을 기준점으로 설정
- BFS 또는 DFS를 이용하여 흰색이 아닌 영역에 도달할 때까지 일괄 삭제
- 외곽에서부터 안쪽으로 탐색하므로 오브젝트 내부의 흰색은 보존됨

### 2단계: 경계면 알파 처리

단순 삭제만으로는 경계가 지저분해지는 문제가 있다. 이를 해결하기 위해 경계면 픽셀에 흰색 비례 투명도를 적용한다.

- 경계면에서 안쪽으로 일정 범위(1~2픽셀 등 조절 가능)의 픽셀을 대상으로 처리
- 해당 픽셀의 밝기(흰색에 가까운 정도)에 비례하여 알파(투명도) 값을 부여
  - 흰색에 가까울수록 → 높은 투명도 (알파 낮음)
  - 오브젝트 본래 색에 가까울수록 → 낮은 투명도 (알파 높음)
- 이를 통해 자연스러운 페더링 효과를 구현하여 깔끔한 경계를 만든다

## 동작 모드

### 모드 1: 단순 모드 (마스크 불필요)

- 이미지 가장자리(외곽)를 탐색 시작점으로 사용
- BFS/DFS로 외곽에서부터 흰색 영역을 탐색하여 삭제
- 오브젝트 내부에 흰색(흰 옷, 하이라이트 등)이 있더라도 경계면에 의해 차단되므로 영향 없음
- 별도의 입력 없이 이미지만으로 처리 가능

### 모드 2: 마스크 모드 (복잡한 경우)

- 경계면이 외딴 섬처럼 독립적으로 존재하는 경우에 대응
- 외곽 탐색만으로는 도달할 수 없는 내부 배경 영역이 존재할 때 사용
- 별도의 마스크 파일을 입력으로 제공
- 마스크 파일이 지정하는 영역을 기준으로 동일한 알고리즘(탐색 + 알파 처리)을 적용

## 기대 효과

- 흰색 배경 기반이라는 단순한 전제 하에 복잡한 알파 매팅 없이 자연스러운 배경 제거 가능
- 경계면 알파 처리로 안티앨리어싱 아티팩트 최소화
- 마스크 모드를 통해 복잡한 구조의 이미지에도 유연하게 대응
